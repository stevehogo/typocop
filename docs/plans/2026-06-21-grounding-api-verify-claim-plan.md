# Grounding API — `verify_claim` (anti-hallucination)

## Goal
Add a read-only MCP tool where an agent submits a structured **claim** about the codebase and gets back **verdict (confirmed / refuted / uncertain) + confidence + evidence/counterexample** — so agents stop acting on false beliefs ("nothing calls X, safe to delete"). Operationalizes Feature 1 of `docs/brainstorm/2026-06-20-agent-innovation-features-brainstorm.md` (locked decisions D2/D4/D5; OQ3 lean). Read-only orchestration of machinery typocop already has — **no new infra**.

## v1 claim classes (D4)
**usage/dead-code** ("X has no callers" / "X is dead") · **edge-existence** ("X calls/imports/inherits Y") · **reachability/independence** ("X can reach Y" / "changing X can't affect Y"). Architecture-membership deferred.

## Notes
- **Read-only orchestration, no new infra** — reuses `resolveSymbol`, CALLS/IMPORTS/INHERITS edges, `findDependents`/`findDeadCode`, clamped reachability (`trace-path`), and `explainability` confidence. Many of these landed this session (D2/D3/D6).
- **Honest-uncertainty is mandatory (D5)** — the whole point is not to relocate hallucination into the tool; unprovable/dynamic relationships are `uncertain` with a reason.
- **OQ3 = include the true answer** on a refute (e.g. the actual caller set / the reaching path), not just "false".
- **Layering** (dependency-cruiser): claim logic in `application/querying/`, MCP wiring in `apps/mcp-server/`, no infra siblings.
- **Perf/reliability:** reuse `withQueryTimeout` + `MAX_TRAVERSAL_DEPTH`/branching clamps; sub-second to a few seconds; graceful degradation everywhere (never throw to the agent).
- **Status:** the brainstorm marks this design-stage ("no implementation until design validated"); this is the implementation breakdown built on its locked decisions — plan only, no code yet.
